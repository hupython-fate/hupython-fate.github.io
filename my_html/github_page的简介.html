什么是 GitHub Pages？

免费托管

GitHub Pages 是一项静态站点托管服务，直接从 GitHub 仓库获取 HTML、CSS 和 JavaScript 文件，通过构建过程运行这些文件，并发布网站。

全球访问

部署在 GitHub Pages 上的网站可以通过公共互联网访问，您可以选择使用 GitHub 提供的免费域名或使用自己的自定义域名。

版本控制

利用 Git 版本控制系统的强大功能，您可以跟踪更改、回滚到早期版本，并与他人协作开发您的网站。

如何开始使用 GitHub Pages

1

创建 GitHub 账户

如果您还没有 GitHub 账户，请先 注册一个免费账户。

2

创建新仓库

登录 GitHub 后，创建一个新的仓库。仓库名称格式应为 username.github.io，其中 username 是您的 GitHub 用户名。

仓库名称示例: johnsmith.github.io
3

上传 HTML 文件

将您的 HTML 文件和其他资源（CSS、JS、图片等）上传到仓库中。确保主页面命名为 index.html。

文件结构示例:
index.html
css/style.css
js/script.js
images/logo.png
4

启用 GitHub Pages

在仓库设置中，找到 "Pages" 部分，选择源分支（通常是 main 或 master），然后保存。

5

访问您的网站

几分钟后，您的网站将在 https://username.github.io 上线！

GitHub Pages 功能特性

支持的技术

  • 静态 HTML、CSS 和 JavaScript
  • Jekyll 静态站点生成器
  • 自定义域名支持
  • HTTPS 加密
  • 自定义 404 页面

使用限制

  • 仓库大小限制 1GB
  • 每月带宽限制 100GB
  • 每小时构建限制 10 次
  • 仅支持静态内容
  • 不支持服务器端代码

GitHub Pages 优点

  • 完全免费使用
  • 与 GitHub 工作流无缝集成
  • 自动部署和版本控制
  • 支持自定义域名
  • 全球 CDN 加速

GitHub Pages 限制

  • 仅支持静态网站
  • 不支持服务器端语言
  • 构建时间可能较长
  • 无法处理表单数据
  • 某些高级功能受限

常见问题解答

GitHub Pages 是免费的吗？

是的，GitHub Pages 对公共仓库完全免费。如果您想为私有仓库使用 GitHub Pages，需要 GitHub Pro 账户。

我可以使用自定义域名吗？

是的，您可以将自定义域名指向您的 GitHub Pages 网站。只需在仓库设置中添加自定义域名，并在您的域名注册商处配置 DNS 记录。

GitHub Pages 支持 PHP 或其他服务器端语言吗？

不支持。GitHub Pages 仅支持静态内容（HTML、CSS、JavaScript）。如果您需要服务器端功能，可以考虑使用其他服务如 Heroku、Netlify Functions 或 Vercel。

我的网站上线需要多长时间？

通常，在启用 GitHub Pages 后，您的网站会在 1-10 分钟内上线。后续的更改通常会在您推送到仓库后的 1-5 分钟内更新。
